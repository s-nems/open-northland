# Build desktop release artifacts in CI

**Area:** desktop/tooling · **Origin:** desktop-packaging branch 2026-07-16 · **Priority:** P2

`packages/desktop` builds installers locally (`npm run desktop:dist`), but releases need a CI
pipeline so users can download artifacts without trusting a dev machine — and so SignPath signing
(docs/tickets/tooling/desktop-code-signing.md) has a public build provenance to sign.

## Scope

- A release workflow (tag-triggered) that builds per-OS artifacts with electron-builder from
  `packages/desktop/electron-builder.yml`: Windows NSIS + portable, macOS dmg (unsigned for now),
  Linux AppImage. Windows NSIS/portable cross-build from the Linux runner works; prefer native
  runners per OS if artifacts misbehave.
- Upload artifacts to the GitHub release; embed the version from the tag (root package.json version
  bump or `--config.extraMetadata.version`).
- An app icon: electron-builder currently falls back to the default Electron icon. Needs an
  original OpenNorthland design (legal: no original game art/logos — docs/LEGAL.md) at
  512×512 (icns/ico generated by electron-builder from a PNG in `packages/desktop/build/`).
- Auto-update is a separate later decision (electron-updater needs a publish provider); do not wire
  it here beyond not blocking it.

## Verify

CI run on a tag produces installable artifacts for all three OSes; the Windows portable exe writes
to `portable-data/` next to itself when that dir exists (packaged-build check of the marker-dir
behavior, which local dev runs cannot exercise).
